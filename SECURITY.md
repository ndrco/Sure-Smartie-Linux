# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for security-sensitive problems.

Instead, contact the maintainers privately and include:

- a short description of the issue;
- affected versions or commit range if known;
- reproduction steps or proof of concept;
- impact assessment.

Contact: NDRco <ndrco@yahoo.com>

## Scope

Security-relevant reports may include:

- privilege escalation in the GUI helper flow;
- unsafe file writes or service management behavior;
- plugin loading vulnerabilities;
- malformed config handling that can lead to unsafe system behavior.
